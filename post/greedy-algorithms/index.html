Greedy Algorithms

  • 区间贪心问题（一般按先按左端点或者右端点排序） e.g:区间重复点问题，给定N个区间，然后选尽可能少的点，使得每个区间至少包含一个被选的点
    1. 思路：按右端点排序，每层吃线段最右边的端点，遍历过程中如果包含上一次选的点就pass。
#include<bits/stdc++.h>
using namespace std;
const int N = 10010;
struct node{
    int l,r;
    bool operator<(const node &w){
        return r<w.r;//按右端点从小到大排序
    }
}range[N];
int n;
void solve()
{
    cin >> n;
    for(int i=0;i<n;++i){
        int l,r;
        cin >> l >> r;
        range[i] = {l,r};
    }
    int ed = -2e9,cnt = 0;
    sort(range,range+n);
    for(int i=0;i<n;++i){
        if(range[i].l>ed) {
            ++cnt;
            ed = range[i].r;
        }
    }
    cout << cnt <<endl;
}

int main()
{
   solve();
    return 0;
}
  • Huffman tree e.g:合并果子 合并n堆果子（每次合并两堆，消耗的体力为两堆重量和）求合并消耗的最小体力 本质上是一棵完全二叉树，每一个叶子节点到根节点的权值和为 路径长度*叶节点值
  • 思路： 最小的两个点，深度一定最深，且可以互为兄弟。（小的东西让他累积的多，大的深度小就可以让全局最小） 核心思路就是每次选最小的两堆合并，然后合并成的一大堆果子入小根堆，继续合并
#include<bits/stdc++.h>
using namespace std;

void solve()
{
   int n;
   cin >> n;
   priority_queue<int,vector<int>,greater<int> >pq;
   while(n--){
    int x;
    cin >> x;
    pq.push(x);
   }
   int ans = 0;
   while(pq.size()>1){
        int a = pq.top();pq.pop();
        int b = pq.top();pq.pop();
        ans += (a+b);
        pq.push(a+b);
   }
   cout << ans <<endl;
}

int main()
{
   ios_base::sync_with_stdio(false);
   cin.tie(0),cout.tie(0);
   solve();
    return 0;
}

有关Huffman tree的一些总结：

  • 实现哈夫曼树的方式有很多种，可以使用优先队列（Priority Queue）简单达成这个过程，给与权重较低的符号较高的优先级（Priority），算法如下： ⒈把n个终端节点加入优先队列，则n个节点都有一个优先权Pi，1 ≤ i ≤ n ⒉如果队列内的节点数>1，则： ⑴从队列中移除两个最小的Pi节点，即连续做两次remove（min（Pi）, Priority_Queue) ⑵产生一个新节点，此节点为（1）之移除节点之父节点，而此节点的权重值为（1）两节点之权重和 ⑶把（2）产生之节点加入优先队列中 实际应用中可以用来进行数据压缩（出现几率大的（权值高）放在较浅的层，低的则放在深层，则带权路径和最短）

$Leetcode .910 最小差值II（贪心的好题）$

  • https://leetcode.cn/problems/smallest-range-ii/description/?envType=daily-question&envId=2024-10-21
  • 思路
    1. 该贪心满足一定的规律，具有一定的单调性
    2. 一定是从某个点开始全都是加或全都是减k，极端情况：全都加或全都减，则最大和最小差值不变。
    3. 一般情况分为两种：1. 从i这个下标之前全部为-k，从i这个下标之后全是+k。 2. 从i这个下标之前全是+k，从i这个下标之后全是-k 4.现在考虑case1，此时一眼看出，最小的仍然是0这个下标对应的值 = nums[0] - k,而最大则变为，nums.back()+ k ,则差值恒为nums.back()-nums[0]+2k,不减反增；再考虑case2：前面全+k，后面全-k，则最大值的可能情况有两种，本质上变成了断点在 i 的分段函数最大值最小值问题，
      1. nums[i]-k < nums[i+1]-k < … < nums.back()-k
      2. nums[0]+k < nums[1]+k < … < nums[i-1]+k 这俩分段函数最大值可能是 nums.back() - k 也有可能是 nums[i-1] + k. 最小值可能是 nums[0] + k 也有可能是 nums[i-1] - k
        所以很轻松能够得到贪心思路，遍历每种可能的i，然后记录当前最大值和最小值，然后更新两者之差的最小值
    class Solution {
public:
    int smallestRangeII(vector<int>& nums, int k) {
        sort(nums.begin(),nums.end());
        const int n = nums.size();
        int ans = nums.back()-nums[0];
        for(int i=1;i<n;++i){
            int mx = max(nums[i-1]+k,nums.back()-k);
            int mn = min(nums[0]+k,nums[i]-k);
            ans = min(ans,mx-mn);
        }
        return ans;
    }
};
